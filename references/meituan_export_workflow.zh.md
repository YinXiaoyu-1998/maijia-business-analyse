# 美团管家自助取数导出流程

用于从美团管家报表中心取得本 skill 所需的 `营业分组表` `.xlsx` 源文件；当经营分析需要菜品穿透时，也用于取得 `菜品主题数据` `.xlsx` 源文件。

## 浏览器前提

- 使用已经登录美团管家的 Chrome 或其他浏览器会话。
- 如需 agent 操作浏览器，优先使用能复用用户登录态的浏览器控制工具。
- 不要在对话或日志里暴露下载后的业务明细行；只报告文件路径、日期范围、行数和校验结果。

## 文件命名规范

所有从美团下载的原始 `.xlsx` 文件统一保存到 `documents/raw_exports/`。不要只保留美团自动生成的长文件名；下载后按以下规则重命名：

| 数据类型 | 标准文件名 |
|---|---|
| 营业分组表 / 自助营业取数 | `maijia_business_YYYYMMDD_YYYYMMDD.xlsx` |
| 菜品主题数据 / 自助菜品取数 | `maijia_dishes_YYYYMMDD_YYYYMMDD.xlsx` |
| 菜品库基础信息 / 菜品导出 | `maijia_dish_catalog_YYYYMMDD.xlsx` |

如果同一日期范围因为平台限制拆成多份，在扩展名前加分片号：`_part01`、`_part02`。例如 `maijia_business_20250301_20250620_part01.xlsx`。

## 数据取得原则：不要复用历史下载清单

开始取数前，先检查本地 `documents/raw_exports/` 是否已经有按命名规范保存的目标文件。若文件存在，优先校验文件类型、zip 完整性、日期范围和必要字段；确认无误后再复用。

如果本地没有合格文件，不要一上来就去美团网页的 `下载清单` 查找历史下载记录。历史记录可能来自错误日期、错误报表、未选全字段或旧版本导出，不能作为可靠数据源。

正确流程是：先进入对应的取数/导出页面，设置目标日期范围和全部必要字段，点击 `查询` 并确认页面数据范围，然后点击 `导出` 创建新的导出任务。只有在新任务创建之后，才进入 `下载清单` 下载刚创建的记录，并用报表名、日期范围、申请时间/更新时间共同核对。

## 操作步骤：营业分组表

1. 打开 `https://pos.meituan.com/web/report/main#/rms-report/home`。
2. 点击左侧导航 `自助取数`。
3. 在展开菜单里点击 `自助营业取数`。
4. 在 `常用查询` 中选择 `全量数据`。
5. 设置 `营业日期`：
   - 用户说“过去 7 天”时，优先使用已经完整结束的 7 个营业日，避免把当天未完结数据混入。
   - 例如当前日期为 `2026-06-14`，则选择 `2026/06/07 - 2026/06/13`。
6. 点击 `展开筛选`。
7. 勾选所有参数；通常 `全量数据` 会自动勾选全部字段，但仍要用截图或页面状态确认。
8. 点击 `查询`，等待表格刷新并确认第一行日期落在目标范围内。
9. 点击右上角 `导出`。
10. 弹窗提示导出任务创建后，点击弹窗中的 `前往下载清单`。
    - 如果当前页没有明显跳转，检查浏览器是否新开了 `...#/rms-report/promiseDownload` 的下载清单标签页。
    - 也可以从顶部 `导入导出 -> 下载清单` 进入同一页面。
11. 在 `下载清单记录` 中找到刚才日期范围对应的记录。
    - 用 `申请内容` 核对报表名与营业日期，例如 `营业分组表(营业日期【2026/06/09-2026/06/15】)`。
    - 用 `申请时间` / `更新时间` 核对是否是刚刚创建的任务。
    - 等待 `状态` 为 `导出完成`；如未完成，点击右上角 `刷新`。
12. 点击该记录最右侧 `操作` 列里的 `下载`。
    - 注意这里是表格行右侧、通常紧挨 `删除` 的橙色 `下载` 文本，不是报表页右上角的 `导出` 按钮。
    - 如果有多条记录，优先下载日期范围和申请时间都匹配的那一行。

## 操作步骤：菜品主题数据

当用户需要“菜品完整信息”“穿透到菜品”“菜品/档口归因”或需要文件名类似 `maijia_dishes_YYYYMMDD_YYYYMMDD.xlsx` 时，使用 `自助菜品取数`。

1. 打开 `https://pos.meituan.com/web/report/main#/rms-report/home`。
2. 点击左侧导航 `自助取数`。
3. 在展开菜单里点击 `自助菜品取数`。
4. 设置日期范围：
   - 优先使用已经完整结束的营业日。
   - 如果今天是 `2026-06-25`，需要最近完整日时，优先选择 `2026/06/24`。
5. 点击 `展开筛选`。
6. 勾选所有字段组：
   - `查询维度`
   - `菜品销量信息`
   - `菜品关联信息`
   - `菜品成本及服务`
7. 点击 `查询`，等待表格刷新，确认日期范围和记录数。
8. 点击右上角 `导出`。
9. 弹窗提示导出任务创建后，点击 `前往下载清单`。
10. 在 `下载清单记录` 中找到刚才日期范围对应的记录。
    - 用 `申请内容` 核对报表名与日期，例如 `菜品主题数据(日期【2026/06/24-2026/06/24】)`。
    - 用 `申请时间` / `更新时间` 核对是否是刚刚创建的任务。
    - 等待 `状态` 为 `导出完成`；如未完成，点击右上角 `刷新`。
11. 点击该记录最右侧 `操作` 列里的 `下载`。
12. 保存到 `documents/raw_exports/` 目录，并按文件命名规范重命名，例如 `documents/raw_exports/maijia_dishes_20260614_20260620.xlsx`。

导出的 `自助菜品取数` Excel 常见结构：

- 第 1 行：报表标题，如 `菜品主题数据`。
- 第 2 行：筛选条件说明，含日期范围和已选字段组。
- 第 3 行：真实字段表头。
- 第 4 行起：数据行。

处理该文件时应跳过前两行，把第 3 行作为表头。部分美团导出的 `.xlsx` 会把 OOXML worksheet dimension 写成 `1 x 1`，导致 `openpyxl` 的 `max_row` / `max_column` 误判；遇到这种情况时，应流式遍历行或解析 `xl/worksheets/sheet1.xml`，不要依赖维度元数据。

## 操作步骤：菜品库基础信息

当用户需要“档口归因”“基础分类归因”“菜品库”“最新菜品基础信息”，或需要把 `自助菜品取数` 与稳定菜品分类映射起来时，使用 `运营中心` 的 `菜品库` 导出。这个流程不是报表中心自助取数；它提供菜品维度表。

1. 打开 `https://pos.meituan.com/web/operation/main#/`，或在已登录页面点击顶部 `运营中心`。
2. 点击左侧导航 `菜品管理`。
3. 在弹出的菜单里点击 `菜品库`。
4. 在左侧品牌树中选择目标品牌：
   - 默认选择 `麦家小馆`。
   - 如果用户要求其他子品牌，选择对应品牌。
5. 点击页面右上角 `菜品导出`。
6. 在导出弹窗里选择 `导出菜品基础信息`。
7. 选择 `全部字段`，确保导出包含 `基础分类`、`打印出品档口`、`出品部门`、`设置出品部门`、`预估成本` 等字段。
8. 点击右下角 `确定`。
9. 如果 macOS 出现保存弹窗，保存到 `documents/raw_exports/` 目录，并按文件命名规范重命名，例如 `documents/raw_exports/maijia_dish_catalog_20260626.xlsx`。

导出的菜品库 Excel 常见结构：

- `总部菜品`：总部菜品基础信息。第 1 行通常是标题，第 2 行是筛选/说明，第 3 行是真实字段表头，第 4 行起是数据。
- `总部套餐`：套餐与套餐内菜品的组合信息。用于把套餐销售拆到组成菜品；如果不拆解，应把套餐单独列为套餐口径。

字段使用规则：

- `档口 = 基础分类`。在经营分析和周会报表中，用 `总部菜品` 的 `基础分类` 作为档口/摊位/品类经营归因维度。
- `打印出品档口`、`出品部门`、`设置出品部门` 更偏厨房出品或打印路由，不默认替代管理口径的档口。
- 优先用菜品编码 / SKU 编码与 `自助菜品取数` 做关联；如果销售表没有稳定编码，再用标准化后的 `菜品名称` + `规格名称` 近似关联，并在报告中提示可能存在重名或套餐拆分误差。

## Chrome 拦截时的处理

有时点击下载清单行右侧的 `下载` 会新开或跳转到 `s3plus.sankuai.com` 临时 URL，但被 Chrome 扩展或客户端拦截，页面显示 `ERR_BLOCKED_BY_CLIENT`。

处理方式：

1. 从地址栏复制完整临时下载 URL。
2. 使用脚本下载：

```bash
python3 scripts/download_meituan_signed_url.py \
  --url '<signed-s3plus-url>' \
  --output documents/raw_exports/maijia_business_YYYYMMDD_YYYYMMDD.xlsx
```

3. 下载后校验：

```bash
file path/to/export.xlsx
unzip -t path/to/export.xlsx
```

应看到 `Microsoft OOXML` 且 `No errors detected`。
