# 麦肯锡风格经营诊断报告口径

## 报告定位

输出应是面向老板和经营管理层的经营诊断，不是原始数据看板。采用“先结论、后证据、再行动”的结构：

1. 执行摘要：3-5 条 answer-first 经营判断。
2. 数据与口径：说明周期、行数、门店数、核心字段、不可回答的问题。
3. 整体经营画像：收入、订单、客单、折扣和收入质量。
4. 门店组合诊断：门店排名、分型、异常与复制机会。
5. 渠道结构分析：店内、外卖、自提、平台渠道的规模与质量。
6. 餐段/时段机会：晚餐、午餐、夜宵等时段收入热力。
7. 机会池与行动路径：30/60/90 天动作，标注证据强弱和需补充数据。

## 计算原则

- 对收入、订单、优惠、退款等金额/数量字段先加总。
- 对折扣率、客单价、退款率、会员占比等比率字段，必须在聚合后重新计算。
- 不要简单平均 `开台率`、`翻台率` 等运营比率，除非能确认权重口径。
- 没有成本、人工、租金、平台佣金、菜品毛利时，不要写“利润提升”或“亏损根因”的确定性结论。

## 输出风格

- HTML 报告优先使用图表、KPI 卡片、排序表、热力图和少量结论短句。
- 避免大段文字和原始数字堆砌。
- 结论必须能回溯到事实表或 `analysis_summary.json`。
- 建议动作要明确到经营抓手，例如折扣治理、低客单修复、外卖渠道治理、会员渗透补齐。
