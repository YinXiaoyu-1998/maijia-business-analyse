# 麦家小馆营业数据经营诊断报告分析蓝图

## 1. 报告定位

本报告定位为一份面向老板和经营管理层的「麦肯锡风格经营诊断报告」，不是单纯的数据看板，也不是完整利润审计。

核心问题：

> 麦家小馆在 2025/05/26-2026/05/25 期间，哪些门店、渠道、餐段和消费结构真正贡献了收入？哪些地方存在收入质量、折扣、客单、翻台、会员、外卖结构或运营效率问题？管理层接下来应该优先抓哪几个经营杠杆？

报告原则：

- 先结论后证据：每章用一句可讨论的经营判断开头。
- 事实驱动：所有定量结论必须能回溯到营业数据字段和计算口径。
- 结构化拆解：按「整体 -> 门店 -> 渠道 -> 时段/餐段 -> 折扣/会员/效率 -> 机会」推进。
- 行动导向：最终落到 30/60/90 天可执行动作，而不止描述现象。
- 谨慎边界：没有成本、租金、人工、菜品、评价、商圈数据时，不做利润和根因的过度断言。

## 2. 源数据概况

源文件：`documents/business_data.xlsx`

工作表：`营业分组表`

已确认结构：

- 第 1 行：报表标题。
- 第 2 行：导出筛选条件。
- 第 3 行：字段名。
- 第 4 行起：营业分组数据。
- 字段数量：152 列，最后一列为 `EV`。
- 导出日期范围：`2025/05/26-2026/05/25`。

基础维度字段：

| 字段 | 用途 |
|---|---|
| 营业日期、周、月 | 趋势、周期、季节性、月度经营节奏 |
| 门店名称、城市、商户号 | 门店层级分析、门店排名、门店分型 |
| 一级/二级/三级/四级组织机构名称 | 组织层级汇总 |
| 订单分类、订单来源 | 渠道与订单结构 |
| 时段、餐段 | 日内时段和餐段机会分析 |
| 退单类型名称 | 正常单/退单结构 |
| 是否是会员 | 会员与非会员对比 |
| 就餐方式 | 堂食、外带、外卖/未知等结构 |

## 3. 指标计算原则

### 3.1 可直接加总的指标

这些字段在聚合时可以按维度直接求和：

- 收入类：`营业额(元)`、`订单营业收入`、`优惠金额`
- 订单类：`有效订单量`、`逆向订单量`、`正向订单量`、`已结账订单量`
- 消费类：`消费次数`、`消费桌数`、`用餐人数`
- 店内类：`店内营业额`、`店内营业收入`、`店内优惠金额`、`店内订单量`、`店内正向单订单量`、`店内退单量`、`店内已结账订单量`、`店内退款金额`
- 外卖类：`外卖营业额`、`外卖营业收入`、`外卖折扣金额`、`外卖订单量`、`外卖正向单订单量`、`外卖退单量`、`外卖已结账订单量`、`外卖退款金额`
- 平台外卖：美团、京东、饿了么对应的营业额、收入、折扣、订单、退款字段
- 自提类：自提营业额、收入、折扣、订单、退款字段
- 支付类：现金、扫码支付、微信、支付宝、团购、抵用券、优惠券、自定义记账等金额和次数字段
- 会员类：`会员消费金额`、`会员订单收入`、`店内营销会员订单量`、`店内非会员订单量`、`外卖营销会员订单量`、`店内营销会员营业额`、`店内非会员营业额`、`店内营销会员营业收入`、`外卖营销会员营业额`
- 营业类：`营业天数`

### 3.2 聚合后必须重新计算的指标

这些字段不能简单求和，必须在目标聚合层级重新计算：

| 指标 | 推荐口径 |
|---|---|
| 折扣率 | `优惠金额 / 营业额(元)`，若营业额为 0 则置空 |
| 折前单均 | `营业额(元) / 正向订单量` |
| 折后单均 | `订单营业收入 / 正向订单量` |
| 店内折前单均 | `店内营业额 / 店内正向单订单量` |
| 店内折后单均 | `店内营业收入 / 店内正向单订单量` |
| 外卖折前单均 | `外卖营业额 / 外卖正向单订单量` |
| 外卖折后单均 | `外卖营业收入 / 外卖正向单订单量` |
| 平台外卖单均 | 对应平台营业额/收入除以对应平台正向订单量 |
| 自提单均 | 自提营业额/收入除以自提正向订单量 |
| 退款率 | `退款金额 / 营业额` 或 `退单量 / 订单量`，按章节说明 |
| 会员收入占比 | `会员订单收入 / 订单营业收入` |
| 会员订单占比 | `会员订单量 / 正向订单量`，需按可用字段选择 |
| 人均消费 | `订单营业收入 / 用餐人数` |
| 桌均收入 | `店内营业收入 / 消费桌数` |
| 翻台率 | 优先使用源字段作展示；聚合层级若重算，需明确口径 |
| 开台率 | 优先使用源字段作展示；聚合层级若重算，需明确口径 |

### 3.3 需要特别谨慎的字段

- `开台率`、`翻台率`：如果源数据已按很细维度拆分，简单平均会失真。门店/月级展示时应使用加权平均或仅作为参考。
- `桌台数量`、`桌台数x营业天数`：可用于桌效估算，但需确认是否每天固定、是否按门店重复出现在多条分组记录里。
- `是否是会员`：样本中为 `0/1`，需在报告中翻译为非会员/会员。
- `退单类型名称`：如果样本多数为正常单，仍需完整扫描后确认是否有退单类型。
- 支付字段：支付方式可能存在组合支付，支付次数和订单量不一定一一对应。

## 4. 报告章节设计

### 4.1 执行摘要

目标：让老板在 1 页内看到最重要的经营判断和行动优先级。

代码产出：

- 全年总营业额、订单营业收入、优惠金额、折扣率
- 正向订单量、已结账订单量、折前/折后单均
- 门店数、活跃月份、活跃营业天数
- Top/Bottom 门店、渠道、餐段
- 3-5 个最大异常或机会点

LLM 产出：

- 3-5 条 answer-first 结论
- 每条结论的证据压缩
- 管理层优先动作排序

建议图表：

- KPI 卡片
- 经营诊断结论页
- 机会优先级矩阵

### 4.2 数据与口径说明

目标：建立可信度，说明报告能回答什么、不能回答什么。

代码产出：

- 数据周期
- 门店数量
- 城市/组织/渠道/餐段枚举
- 字段完整性、空值、异常值概览
- 是否存在重复或疑似总计行

LLM 产出：

- 口径解释
- 数据局限说明
- 对老板的阅读提醒

建议图表：

- 数据覆盖概览表
- 字段分组说明表

### 4.3 整体经营画像

目标：判断整体经营规模、节奏和收入质量。

代码产出：

- 月度营业额、订单收入、优惠金额、订单量、客单价、折扣率
- 周度趋势作为辅助
- 工作日/周末对比，如日期可解析
- 收入与订单量的峰谷月份

LLM 产出：

- 判断整体趋势是增长、波动、季节性，还是结构性分化
- 解释收入、订单、客单、折扣之间的关系

建议图表：

- 月度收入趋势折线图
- 营业额/优惠/订单收入瀑布图
- 订单量与客单价双轴图

### 4.4 增长拆解

目标：回答收入来自哪里、变化由什么驱动。

代码产出：

- 按月/门店/渠道拆分营业额贡献
- 收入贡献 = 门店数、营业天数、订单量、折后单均、折扣率的组合变化
- Top 贡献门店与拖累门店
- Top 贡献渠道与拖累渠道

LLM 产出：

- 识别主要增长引擎
- 区分规模增长和质量增长

建议图表：

- 收入贡献瀑布图
- 门店贡献 Pareto 图
- 渠道贡献堆叠图

### 4.5 门店组合诊断

目标：把门店分成可管理的几类，而不是只给排名。

代码产出：

- 门店营业额排名
- 门店订单收入、正向订单量、折后单均、折扣率、用餐人数
- 店内/外卖/自提收入结构
- 桌效、翻台、开台、消费桌数
- 异常指标：高折扣、低客单、高退款、低效率

建议门店分型：

| 分型 | 判定逻辑 |
|---|---|
| 明星店 | 高收入、高订单、高客单或高效率，折扣可控 |
| 稳定现金流店 | 收入中高、波动低、指标均衡 |
| 结构机会店 | 收入不低，但折扣高、渠道结构偏、会员弱或餐段不均衡 |
| 低效待改善店 | 收入低、订单低、桌效低或退款/折扣异常 |

LLM 产出：

- 给每类门店写经营画像
- 提炼每类门店的管理动作
- 避免把单一指标排名误读为综合好坏

建议图表：

- 门店收入/折后单均四象限
- 门店收入/折扣率散点图
- 门店经营分层表

## 5. 代码与 LLM 分工

### 5.1 代码必须完成的工作

- 流式读取大体积 xlsx，避免一次性整体加载。
- 校验表头和必要字段。
- 标准化日期、数字、会员标记、渠道字段。
- 输出聚合后的事实表，而不是让 LLM 直接读原始明细。
- 计算所有 KPI、排名、分布、异常、机会测算。
- 生成可复核的中间文件，例如 CSV/Parquet/JSON 摘要。
- 生成报告所需图表数据。

建议中间数据表：

| 表名 | 粒度 | 用途 |
|---|---|---|
| `overall_kpis` | 全局 | 执行摘要 |
| `monthly_trend` | 月 | 整体趋势 |
| `store_summary` | 门店 | 门店分型 |
| `channel_summary` | 渠道/订单分类/订单来源 | 渠道诊断 |
| `daypart_summary` | 餐段/时段 | 餐段机会 |
| `store_daypart_matrix` | 门店 x 餐段 | 门店运营机会 |
| `discount_summary` | 门店/渠道/餐段 | 折扣治理 |
| `member_summary` | 会员标记/门店/渠道 | 会员诊断 |
| `ops_efficiency` | 门店 | 桌效与翻台 |
| `refund_anomalies` | 门店/渠道/日期 | 风险清单 |
| `opportunity_sizing` | 机会项 | 行动优先级 |

### 5.2 LLM 应该完成的工作

- 从聚合事实中提炼经营洞察。
- 按重要性排序，而不是平铺所有指标。
- 写出麦肯锡式页面标题：标题本身就是结论。
- 生成管理层能讨论的假设。
- 把发现转化为行动建议。
- 标注不确定性和需要业务验证的问题。

LLM 输入不应是原始大表，而应是结构化摘要：

```json
{
  "overall_kpis": {},
  "top_findings": [],
  "store_segments": [],
  "channel_findings": [],
  "daypart_findings": [],
  "discount_findings": [],
  "member_findings": [],
  "opportunity_sizing": []
}
```

LLM 输出建议结构：

```json
{
  "executive_summary": [
    {
      "headline": "结论型标题",
      "evidence": ["关键数据1", "关键数据2"],
      "implication": "经营含义",
      "recommended_action": "建议动作"
    }
  ],
  "chapter_narratives": [],
  "open_questions": [],
  "management_actions": []
}
```

## 6. 第一轮实际分析的最小可行版本

为了先跑通，不建议一开始就做完整报告。第一轮只做以下内容：

1. 数据体检：字段、日期、门店、渠道、餐段、异常值。
2. 整体 KPI：收入、订单、折扣、客单、会员、店内/外卖/自提结构。
3. 门店分型：门店收入、客单、折扣、订单、渠道结构四象限。
4. 餐段机会：时段/餐段热力图。
5. 折扣与会员：判断收入质量和会员贡献。
6. 初版机会池：3-5 个可测算经营机会。

第一轮报告交付物：

- 一份结构化数据摘要。
- 一份图表数据包。
- 一份 Markdown 版经营诊断草稿。
- 可选：后续再转成 PPT、PDF 或 Word。

## 7. 现有数据无法直接回答的问题

以下问题不能仅靠 `business_data.xlsx` 得出确定结论：

- 净利润、毛利率、食材成本、人工成本、租金。
- 菜品级爆品/滞销、菜单结构问题。
- 外卖平台佣金后的真实净收入。
- 顾客复购率、会员生命周期、储值情况。
- 商圈客流、竞品、门店面积、座位数变化。
- 员工排班、人效、服务质量。
- 顾客评价、差评原因、品牌认知。

报告中只能把这些作为「待补充数据」或「待验证假设」，不能伪装成已有结论。

## 8. 下一步执行建议

下一步应编写一个只读、流式的数据分析脚本，完成：

1. 从 `business_data.xlsx` 抽取并标准化明细。
2. 生成 `overall_kpis`、`monthly_trend`、`store_summary`、`channel_summary`、`daypart_summary` 等事实表。
3. 输出一个紧凑的 JSON 摘要，供 LLM 生成第一版经营洞察。
4. 同时输出 CSV，便于人工复核和后续做图。

脚本仍然不应一次性把整个 xlsx 工作簿导入内存。
